# Vector Pipeline

This directory contains the dataset-level orchestration layer for the current drawing representation pipeline.

## Directory Layout

```text
vector_pipeline/
  svg_parser/
    svg_parser_v2.py              # SVG -> parser JSON

  tokenizer/
    run_tokenizer.py              # parser JSON -> tokenized JSON
    run_embedding.py              # tokenized JSON -> primitive embeddings
    *_tokenizer.py                # channel-specific token rules
    embedding_generator.py        # channel embeddings + 256-dim fusion

  pipeline/
    run_svg_dataset_pipeline.py   # batch SVG dataset pipeline

  outputs/
    processed_dataset/            # generated by the batch script
      parsed/
      tokenized/
      embeddings/
      manifest.json
      summary.json
      embedding_dataset.jsonl
```

## Batch Command

Run a small smoke test first:

```bash
python3 -m vector_pipeline.pipeline.run_svg_dataset_pipeline --input-dir "SVG Dataset" --output-dir outputs/processed_dataset_sample --limit 3 --overwrite
```

Run the full dataset:

```bash
python3 -m vector_pipeline.pipeline.run_svg_dataset_pipeline --input-dir "SVG Dataset" --output-dir outputs/processed_dataset --overwrite
```

Skip final embeddings if you only want parser and tokenizer outputs:

```bash
python3 -m vector_pipeline.pipeline.run_svg_dataset_pipeline --input-dir "SVG Dataset" --output-dir outputs/processed_dataset_tokens --skip-embeddings
```

## Current Embedding Configuration

Each channel dimension is selected from `{16, 32, 64, 128}` and the final fused vector is fixed at 256 dimensions.

```text
type_emb  = 32
geom_emb  = 128
style_emb = 32
pos_emb   = 64
text_emb  = 64
group_emb = 16

z_i = Fuse(type_emb, geom_emb, style_emb, pos_emb, text_emb, group_emb) -> 256
```
